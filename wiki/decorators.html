Decoradores de Seguridad

PyEgar utiliza decoradores personalizados para gestionar la visibilidad de los métodos dentro del Kernel. Esto asegura que la lógica crítica del motor permanezca intacta mientras el usuario desarrolla su juego.

@private
Solo Interno

Este decorador marca una función como exclusiva del Kernel. Si un script de usuario intenta llamar a una función marcada como @private, el centinela ERROREGAS registrará una advertencia de acceso no autorizado.

Propósito: Proteger métodos como _update_all_logic que gestionan la memoria y el renderizado directamente.
@public
Acceso Total

Define métodos que están diseñados para ser usados por el desarrollador. Estos métodos tienen validación de tipos automática para evitar datos corruptos.

@public
def eg_Speed(self, value):
    # El decorador valida que 'value' sea un número
    self.speed = float(value)

Implementación Técnica

Los decoradores de PyEgar utilizan functools.wraps para mantener los metadatos de las funciones originales, facilitando la depuración (Debugging).

¿Cómo funcionan?

  1. Interceptan la llamada a la función.
  2. Consultan el nivel de seguridad en el bus de datos _eg_state.
  3. Si el origen de la llamada es externo y la función es privada, se bloquea la ejecución.
