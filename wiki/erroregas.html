Centinela ERROREGAS

SISTEMA ONLINE

ERROREGAS es el módulo de seguridad activa de PyEgar. Su función es interceptar excepciones, validar la integridad de los nodos inyectados y generar reportes legibles antes de que el motor colapse.


>>> [ERROREGAS] Inicializando vigilancia...
>>> [ERROREGAS] Escaneando decoradores @private...
>>> [ERROREGAS] Advertencia: Delta Time desincronizado en Entity_01
>>> [ERROREGAS] Kernel V0.3 protegido.

Monitoreo Activo

El centinela actúa principalmente en tres frentes del motor:

Validación de Inyección

Verifica que los componentes pasados a libs() sean clases válidas de PyEgar. Si intentas inyectar un objeto no soportado, ERROREGAS detiene el proceso antes de corromper la memoria.

Intercepción de Atributos

Vigila el acceso a variables protegidas. Si un usuario intenta modificar una variable de solo lectura (como el bus de datos _eg_state), se dispara un aviso de integridad.

Inicialización

ERROREGAS es lo primero que se carga al hacer import pyegar. No requiere configuración manual.

# Internamente en el Kernel
try:
    from .Components.ErrorEgar.erroregas import ERROREGAS
    ERROREGAS.start()
except ImportError:
    print("Fallo crítico: El centinela no responde.")
