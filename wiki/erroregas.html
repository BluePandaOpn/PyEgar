Centinela ERROREGAS

KERNEL PROTEGIDO

ERROREGAS es el centinela de seguridad activa de PyEgar. En la V0.5, se encarga de supervisar el arranque del motor y validar que cada inyección de componentes sea segura para evitar el colapso del sistema.


>>> [Kernel V0.5] Compilando módulos | ....
>>> [Kernel V0.5] SISTEMA LISTO PARA RENDERIZAR.
>>> [ERROREGAS] Monitoreando inyección en MemoryGame...
>>> [Node2D] Validación exitosa: CollisionShape2D inyectado.

Funciones de Vigilancia V0.5

Validación de Capa Node2D

Cuando usas libs(self, ...), ERROREGAS escanea los componentes. Si intentas inyectar una clase que no es compatible con el pipeline de renderizado, el centinela genera un reporte azul y detiene la ejecución para proteger el Kernel.

Detección de Recursión Estática

V0.5 incluye un sistema que detecta si un objeto está intentando llamarse a sí mismo en el on_draw (el error común del motor anterior). ERROREGAS corta el hilo de ejecución antes de agotar la memoria del sistema.

Secuencia de Arranque

El centinela se autoinicia durante el import para proteger el motor incluso antes de crear la ventana.

# PyEgar Boot Logic
try:
    from .Components.erroregas import ERROREGAS
    ERROREGAS.start() # Activa el Kernel V0.5
except ImportError:
    print(">>> [PyEgar Warning] Centinela no encontrado.")
