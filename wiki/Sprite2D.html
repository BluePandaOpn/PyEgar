Sprite2D Component

Gestión de texturas, escalado y transformaciones visuales.

¿Qué es Sprite2D?

Es el componente encargado de cargar y mostrar archivos de imagen (PNG, JPG) en el mundo del juego. En el Kernel V0.5, los sprites se integran automáticamente con el Pipeline de Render para optimizar el dibujado.

Flip Automático

El componente lee la variable x_direction del nodo. Si es negativa, el sprite se invierte horizontalmente de forma automática sin que tengas que procesar la imagen manualmente.

Propiedades y Métodos

load_texture(path)

Carga una imagen en la memoria de video. Se recomienda usarlo solo en el __init__.

scale(x, y)

Ajusta el tamaño visual del sprite. Esto no afecta a la CollisionShape2D a menos que se sincronice.

Implementación

Un ejemplo básico de un proyectil que usa una textura:

class Bala:
    def __init__(self):
        libs(self, Sprite2D)
        # Cargamos la imagen desde la carpeta Assets
        self.sprite.load_texture("Assets/bala.png")
        self.sprite.scale(0.5, 0.5)

    def on_draw(self):
        # El Kernel se encarga de posicionarlo según x, y
        render(self)

Relación con el Viewport

Al igual que el DrawingComponent, la posición final del Sprite en pantalla es calculada restando el offset de la Camera2D.
