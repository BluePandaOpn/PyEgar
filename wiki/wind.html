Ventana y Ciclo de Vida

El módulo Wind.py es el encargado de inicializar Pygame, crear la ventana y gestionar el Game Loop. Su característica principal es la detección automática de configuración.

Configuración Automática

PyEgar lee las variables globales de tu archivo principal (__main__). No necesitas pasar parámetros manuales a la función de inicio.

Variables Detectadas:

  • WIDTH: Ancho de ventana (Default: 800).
  • HEIGHT: Alto de ventana (Default: 600).
  • TITLE: Título de la ventana.
  • FPS: Límite de fotogramas por segundo.

El Corazón: Delta Time (dt)

Para evitar que el juego corra más rápido en PCs potentes, Wind.py calcula el tiempo transcurrido entre frames.

Fórmula interna: milli / 1000.0

Este valor se inyecta automáticamente en todos los nodos que usen movimiento físico.

Implementación del Loop

Existen dos funciones clave para poner en marcha el motor:

1. init()

Prepara la pantalla, el reloj (Clock) y el centinela. Retorna un diccionario de configuración.

2. win(config).start()

Crea la instancia de la ventana y arranca el bucle while True interno que gestiona eventos y renderizado.

# Código Real en main.py
from pyegar import *

# Variables de entorno
WIDTH, HEIGHT = 1280, 720
TITLE = "Space Adventure"
FPS = 144

# Ejecución
config = init()
app = win(config)
app.start()

Función render()

Si necesitas actualizar un objeto manualmente fuera del loop automático, Wind.py expone la función render(nodo).

Esta función hace tres cosas críticas:

  1. Sincroniza el Delta Time con el bus de datos del nodo.
  2. Ejecuta el pipeline de actualización de libs.
  3. Dibuja el nodo en la _current_screen global.
