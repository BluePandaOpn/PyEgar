Documentación de Núcleo • Wind.py V0.5

Ventana y Ciclo de Vida

El módulo Wind.py ha evolucionado en la V0.5 para convertirse en el Kernel del motor. No solo crea la ventana, sino que ahora es el responsable de la sincronización temporal y el manejo de escenas.

Configuración Global

PyEgar V0.5 utiliza un sistema de detección automática de variables desde el archivo principal. Esto elimina la necesidad de configurar manualmente el objeto ventana.

Variables Clave:

  • WIDTH: Ancho de ventana (Soportado hasta 4K).
  • HEIGHT: Alto de ventana.
  • TITLE: Título que aparecerá en el Kernel.
  • FPS: Límite de refresco (Recomendado: 60 o 144).

Kernel y Delta Time (dt)

La gran novedad de la V0.5 es la gestión del tiempo real. El Kernel calcula el Delta Time en cada frame para asegurar que el movimiento sea idéntico en cualquier computadora.

Inyección Automática: self._eg_state["dt"]

Este valor es inyectado por el Kernel en todos los nodos que pasan por libs().

Inicialización y Boot

Para arrancar el motor con la nueva protección del centinela, se utiliza la siguiente estructura:

1. init()

Dispara la animación de carga en la terminal y prepara los módulos de seguridad.

2. win(config).start()

Crea la superficie de dibujo y entra en el bucle infinito que procesa la escena activa.

# Ejemplo de arranque V0.5
from pyegar import *

WIDTH, HEIGHT = 1200, 800
TITLE = "Memory Game V0.5"

# El Kernel despierta aquí
config = init()
app = win(config)

# Registrar y arrancar
render("MiEscena")
app.start()

El nuevo render()

En la V0.5, render() tiene una doble función crucial:

  • Cambio de Escena: Si pasas un string, el Kernel cambia la vista principal del juego inmediatamente.
  • Procesamiento de Nodo: Si pasas un objeto, el Kernel ejecuta el pipeline de físicas, colisiones y dibujo sincronizado con la cámara.
