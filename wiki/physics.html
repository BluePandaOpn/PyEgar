Motor Físico

El sistema físico de PyEgar es determinista. Gracias al uso de Delta Time, un objeto se moverá a la misma velocidad real tanto a 30 FPS como a 144 FPS.

Lógica de Movimiento

Cuando inyectas CharacterBody2D, el motor utiliza la lógica interna de move_logic para procesar la posición del nodo en cada frame.

La ecuación del motor:

Nueva Posición = Posición Actual + (Velocidad * Delta Time * Dirección)

Esto asegura que si defines una velocidad de 200, el objeto recorra exactamente 200 píxeles en un segundo real.

Gravedad (Gravity2D)

La gravedad es un componente opcional que puedes inyectar para juegos de plataformas o simulaciones físicas.

Inyección

libs(self, Gravity2D)

Añade aceleración constante en el eje Y.

Variables de Control

  • self.gravity_force: Fuerza de atracción.
  • self.is_on_floor: Booleano que detiene la caída.
# Ejemplo práctico de un personaje con gravedad
class Hero:
    def __init__(self):
        libs(self, CharacterBody2D, Gravity2D)
        self.gravity_force = 15.0 # Ajuste de peso
        
    def update(self):
        if self.is_on_floor:
            self.jump() # Solo salta si toca suelo

Resolución de Colisiones

PyEgar procesa las colisiones en dos pasos mediante el nodo CollisionShape2D:

  1. Sincronización: La hitbox se mueve a las coordenadas x, y del nodo.
  2. Detección: Se comprueba si el Rect de la entidad solapa con otro Rect activo en el bus de datos.

Nota Técnica

Las colisiones en V0.3 son de tipo AABB (Axis-Aligned Bounding Box). No soportan rotación de colisión aún.
