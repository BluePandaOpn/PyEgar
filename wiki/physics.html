Motor Físico

El sistema físico de PyEgar V0.5 es 100% dependiente del tiempo (Delta Time). Esto significa que las físicas se calculan independientemente de la potencia de procesamiento del usuario.

Cinemática y Movimiento

Al usar CharacterBody2D, el motor habilita el procesador _internal_body_update. Este sistema sincroniza los vectores de velocidad con el reloj maestro del Kernel.

Ley de Movimiento V0.5:

distancia = (velocidad * dt)

Donde dt es el Delta Time calculado por Wind.py

Gravedad Inteligente (Gravity2D)

El componente Gravity2D ha sido blindado contra el RecursionError. Ahora, la gravedad se aplica como una fuerza acumulativa que se reinicia al detectar colisión con suelo.

Atributos Dinámicos

  • self.gravity_force: Escala de aceleración (Default: 9.8).
  • self.velocity_y: Vector vertical acumulado.
  • self.is_on_floor: Sensor de contacto.

Uso Correcto

La gravedad debe ser inyectada mediante libs para que el Kernel la procese automáticamente en el pipeline.

# Personaje con físicas de nueva generación
class Player:
    def __init__(self):
        # Inyección múltiple compatible con V0.5
        libs(self, CharacterBody2D, Gravity2D, CollisionShape2D)
        self.gravity_force = 20.0 # Gravedad personalizada
        
    def on_draw(self):
        # El Kernel aplica la gravedad aquí automáticamente
        render(self)

Resolución de Colisiones (AABB)

PyEgar utiliza un sistema de Axis-Aligned Bounding Boxes. En la V0.5, el método is_hover() ha sido optimizado para detectar el puntero del mouse incluso si la cámara se está moviendo.

Pipeline de Colisión:

  1. Update: Se recalculan los límites del Rect base.
  2. Sync: Se traslada la colisión según el scroll de la Cámara.
  3. Check: Se verifica solapamiento con el bus de datos global.

Interacción de Mouse

Para detectar clics en objetos físicos (como en el Juego de Memoria), usa siempre self.eg_collision.is_hover() dentro del on_draw de tu escena.
