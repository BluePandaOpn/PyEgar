Documentación de Núcleo • Render Pipeline V0.5

Pipeline de Renderizado

El Pipeline de la V0.5 es un sistema de procesamiento asíncrono que garantiza que cada nodo se dibuje con precisión temporal milimétrica gracias al Kernel central.

Flujo de Procesamiento V0.5

Cuando el Kernel ejecuta render(), el objeto atraviesa estas etapas críticas en orden descendente:

  1. Kernel Sync: Se inyecta el dt (Delta Time) actual en el bus del nodo.
  2. Z-Order Sort: El motor decide si el objeto va al fondo o al frente (Capa 0 a 100).
  3. Physics Pre-Step: Se resuelven vectores de velocidad y gravedad.
  4. Drawing Inyection: Se procesan las formas geométricas de eg_draw.
  5. Texture Mapping: El Sprite2D se proyecta considerando el escalado y el flip.
  6. Collision Update: La hitbox se bloquea en la posición final tras el renderizado.
  7. User Hook: Se dispara el método on_draw() para lógica personalizada.

Gestión de Escenas

En la V0.5, el Pipeline ya no solo dibuja nodos individuales, sino que gestiona el Scene Buffer. Al llamar a render("NombreEscena"), el motor limpia la memoria anterior y reconstruye el árbol de nodos de la nueva escena.

Nota: El orden de dibujo se basa en el momento de creación del nodo, a menos que se especifique un z_index manual.

Mejoras de Rendimiento

Viewport Clipping

El motor detecta si un nodo está fuera de la cámara y detiene su renderizado gráfico para ahorrar ciclos de CPU.

Texture Caching

Las imágenes cargadas con load_texture se almacenan en un diccionario global; no se vuelven a leer del disco.
