Pipeline de Renderizado

El "Pipeline" es la secuencia de pasos que PyEgar sigue para dibujar un objeto. Este proceso ocurre dentro de la función render(nodo) definida en Wind.py.

Orden de Ejecución Interna

Cada vez que llamas a render(), el motor ejecuta los nodos inyectados en este orden estricto para garantizar la estabilidad del sistema:

  1. Sincronización de Pantalla: El motor vincula el nodo con la superficie actual de Pygame.
  2. Aplicación de Gravedad: Si el nodo tiene Gravity2D, se ajusta la velocidad en Y.
  3. Actualización Body: Se calcula la nueva posición x, y (CharacterBody2D).
  4. Dibujo de Sprite: Se renderiza la textura en las nuevas coordenadas.
  5. Actualización UI: Se procesan y dibujan los Labels si existen.
  6. Sincronización Hitbox: La caja de colisión se mueve al sitio final del objeto.
  7. Custom Draw: Se ejecuta el método on_draw() del usuario.

El Método on_draw()

PyEgar permite al desarrollador añadir lógica de dibujo personalizada al final del pipeline. Esto es ideal para dibujar indicadores de depuración o efectos visuales simples.

class Player:
    def __init__(self):
        libs(self, CharacterBody2D, Sprite2D)

    def on_draw(self):
        # Este código se ejecuta DESPUÉS de que el motor dibuje el sprite
        print(f"Dibujando en: {self.x}, {self.y}")

Optimización Automática

El renderizado de PyEgar implementa técnicas de ahorro de recursos:

Dirty Rects

El motor solo actualiza las áreas de la pantalla donde ha habido cambios de posición, reduciendo el consumo de GPU.

Sprite Flip

La inversión de imagen se calcula en el pipeline basándose en x_direction, eliminando la necesidad de lógica extra por parte del usuario.
