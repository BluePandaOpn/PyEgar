CharacterBody2D

Control cinemático, movimiento por Delta Time y lógica de entidades.

¿Qué es CharacterBody2D?

Es el nodo principal para cualquier objeto que requiera movimiento complejo controlado por el usuario o IA. A diferencia de un nodo estático, el CharacterBody está optimizado para actualizar su posición en cada frame basándose en el Motor Físico.

Independencia de FPS (Delta Time)

En la V0.5, el método mov() integra automáticamente el dt (Delta Time). Esto garantiza que el jugador se mueva a la misma velocidad en una PC de 30 FPS que en una de 144 FPS.

Métodos Principales

eg_Speed(valor)

Establece la velocidad base de movimiento. El Kernel se encarga de aplicarla al vector de dirección.

mov()

Activa la escucha de teclado (WASD/Flechas) y desplaza el nodo. Es ideal para prototipos rápidos.

Ejemplo de Script

Configuración de un personaje básico con seguimiento de Camera2D:

class Player:
    def __init__(self):
        # Inyectamos el cuerpo y el sprite visual
        libs(self, CharacterBody2D, Sprite2D)
        self.eg_Speed(400) # Píxeles por segundo

    def on_update(self):
        # El Kernel procesa el movimiento automáticamente
        self.mov() 

    def on_draw(self):
        # Se dibuja pasando por el Pipeline de Render
        render(self)

Nota sobre Colisiones

Para que un CharacterBody no atraviese paredes, debe tener inyectado también un CollisionShape2D. El Kernel detectará el sólido antes de finalizar el paso de movimiento.
